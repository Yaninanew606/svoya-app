# Техническое задание: Wellness Mini App для женщин 40+
## Telegram Mini App — Персональный фитнес и питание ассистент

---

## 1. Обзор проекта

**Название:** WellnessApp 40+  
**Платформа:** Telegram Mini App (Web App)  
**Аудитория:** Женщины 40 лет и старше  
**Цель:** Персонализированный ассистент по питанию и тренировкам с адаптивной нагрузкой и AI-сопровождением  
**MVP-срок:** 1–2 недели разработки

---

## 2. Технологический стек

### Frontend
- **Framework:** React 18 + TypeScript
- **Styling:** Tailwind CSS
- **Telegram SDK:** `@twa-dev/sdk` (Telegram Web App SDK)
- **State management:** Zustand
- **Routing:** React Router v6
- **Animations:** Framer Motion
- **Icons:** Lucide React
- **Build tool:** Vite

### Backend
- **Runtime:** Node.js 20+
- **Framework:** Express.js или Fastify
- **Database:** PostgreSQL + Prisma ORM
- **AI интеграция:** Anthropic Claude API (`claude-sonnet-4-20250514`)
- **Bot framework:** grammy (Telegram Bot)
- **Auth:** Telegram WebApp initData validation
- **Hosting:** Railway / Render / VPS

## 3. Дизайн и UI

### Экран 2: Анкета (`/questionnaire`)

**Компонент:** `QuestionnaireScreen.tsx`

**Шаги анкеты (wizard, по одному вопросу на экран):**

**Шаг 1 — Возраст:**
- Тип: число (input type="number") или слайдер 40–70
- Валидация: 35–80 лет
- Нельзя пропустить

**Шаг 2 — Цель:**
- Тип: radio-карточки с иконками
- Варианты:
  - 🏃 Тонус и упругость
  - ⚖️ Снижение веса
  - ⚡ Больше энергии
  - 🧍 Осанка и спина
- Множественный выбор разрешён (до 2 целей)

**Шаг 3 — Уровень активности:**
- Тип: radio-карточки
- Варианты:
  - 🛋️ Почти не двигаюсь
  - 🚶 Прогулки, лёгкая активность
  - 🏋️ Тренируюсь 1–2 раза в неделю
  - 💪 Активна регулярно

**Шаг 4 — Время в день:**
- Тип: radio-карточки
- Варианты:
  - ⏱️ До 10 минут
  - 🕐 15 минут
  - 🕑 20 минут
  - 🕒 30+ минут

**Шаг 5 — Ограничения по здоровью:**
- Тип: мультиселект чекбоксы
- Варианты:
  - Нет ограничений
  - Проблемы с коленями
  - Проблемы со спиной
  - Высокое давление
  - Другое (текстовое поле)
- Можно пропустить (кнопка "Пропустить")

**Кнопки:**
| Кнопка | Шаги |
|--------|------|
| "Далее" | Переход к следующему шагу |
| "Пропустить" | Только на шаге 5, пропустить и перейти дальше |
| "Назад" | Возврат к предыдущему шагу (через tg.BackButton) |

**Прогресс:** Прогресс-бар вверху (1/5, 2/5, … 5/5)

**Сохранение:** Данные сохраняются в Zustand + localStorage по мере заполнения

## 5. Адаптивная логика (умное поведение)

### Снижение нагрузки
**Триггеры:**
- Пользователь выбрал "Был тяжёлый день"
- Настроение = 1 (плохо) или 2 (так себе)
- Нажата кнопка "Нужен легче план"
- Нажата кнопка "Мне больно / дискомфорт"

**Действие:**
```
difficulty → 'easy'
duration → min(текущее - 5, 10)
calories → текущие - 100
Добавить: "сегодня просто короткая прогулка — это тоже движение"
```

### Повышение нагрузки
**Триггер:** 3 или более дней подряд:
- mood >= 3
- nutrition = 'yes' или 'partial'
- workout = 'yes' или 'partial'

**Действие:**
```
difficulty → следующий уровень
duration → текущее + 5 (максимум 30)
Добавить: "ты отлично справляешься, пробуем чуть больше"
```

### Щадящий режим (боль/дискомфорт)
**Триггер:** кнопка "Мне больно"

**Действие:**
- Заменить тренировку на 5-минутный комплекс растяжки
- Записать в профиль как "injury flag"
- На следующий день предложить только лёгкие упражнения

### Возврат после пропуска
**Триггер:** Пользователь не открывал приложение 2+ дня

**Действие:**
- Telegram бот отправляет мягкое сообщение (без давления)
- Пример: "Привет 🌿 Всё в порядке? Новый день — новый старт. Твой план ждёт, когда будешь готова."
- Снизить нагрузку плана на 20%

## 10. Аутентификация

1. Frontend получает `window.Telegram.WebApp.initData`
2. Отправляет на backend с каждым запросом в заголовке `X-Telegram-Init-Data`
3. Backend валидирует подпись через HMAC-SHA256 с Bot Token
4. Извлекает `user.id` и находит/создаёт пользователя в БД

```typescript
// backend/src/middleware/auth.ts
function validateTelegramWebApp(initData: string, botToken: string): TelegramUser | null {
  const params = new URLSearchParams(initData);
  const hash = params.get('hash');
  params.delete('hash');
  
  const dataCheckString = [...params.entries()]
    .sort(([a], [b]) => a.localeCompare(b))
    .map(([k, v]) => `${k}=${v}`)
    .join('\n');
  
  const secretKey = crypto.createHmac('sha256', 'WebAppData').update(botToken).digest();
  const expectedHash = crypto.createHmac('sha256', secretKey).update(dataCheckString).digest('hex');
  
  return hash === expectedHash ? JSON.parse(params.get('user')!) : null;
}
```
